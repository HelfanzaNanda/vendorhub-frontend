# Dynamic Form Playground

The Dynamic Form Playground is a developer-only environment designed to preview, test, and debug Form Schemas before integrating them into business modules. It operates completely disconnected from any backend APIs, making it a robust sandbox for frontend schema configuration.

## Folder Structure

```
playground/
├── components/           # UI components for the Playground dashboard
│   ├── DependencyInspector.tsx  # Visualizes cascading/visibility rules
│   ├── LookupInspector.tsx      # Inspects dynamic/static lookup configurations
│   ├── PayloadPreview.tsx       # Live JSON form state export
│   ├── PlaygroundContent.tsx    # Responsive grid and orchestrator
│   ├── PlaygroundHeader.tsx     # Mode toggles and validation trigger
│   ├── PlaygroundSidebar.tsx    # Schema selector navigation
│   ├── SchemaErrorBoundary.tsx  # Catches schema rendering crashes safely
│   └── ValidationSummary.tsx    # Active validation error list
├── mock/                 # Generates random deterministic mock data from schemas
├── registry/             # Centralized list of all available schemas in the app
├── PlaygroundLayout.tsx  # Root wrapper component
└── index.ts              # Public API
```

## Available Features

- **Schema Preview**: Render full business schemas instantly without routing logic.
- **Form Modes**: Switch between `CREATE`, `EDIT`, `READONLY`, and `DISABLED` modes to test UI states.
- **Mock Data**: Auto-generate form payloads based on schema structure to test filled states quickly.
- **Validation Summary**: Manually trigger cross-field and standard validation rules and view interactive errors.
- **Dependency Inspector**: Trace fields that depend on parent values, tracking real-time enable/visible status.
- **Lookup Inspector**: Analyze external API endpoint usage and static option values dynamically.
- **Payload Preview**: Extract and download exact JSON generated by the Engine.

## How to add a new schema

1. Locate `src/modules/dynamic-form-v2/playground/registry/schema-registry.ts`.
2. Import your raw FormSchema.
3. Append it to the exported `SchemaRegistry` array:
   ```typescript
   export const SchemaRegistry: PlaygroundSchema[] = [
     // ... existing schemas
     {
       id: 'my-new-schema',
       title: 'My Custom Schema',
       description: 'Testing new features',
       category: SchemaCategory.VENDOR, // Or MASTER, WORKFLOW, etc.
       schema: MyImportedSchema
     }
   ];
   ```

## How to open the Playground

The playground is mounted under a non-production route:
- Access `/dev/dynamic-form` in your local development environment.
- Do not expose this route in production configurations.
